# Pluely 截图功能使用指南

## 快速开始（面试场景）

### 1. 配置截图功能

打开应用，进入 **设置 > 截图**：

1. **截取方式**：选择"选区模式"（推荐）或"截图模式"
   - 选区模式：拖动选择屏幕区域，更精准
   - 截图模式：一键截取整个屏幕

2. **处理模式**：选择"自动模式"
   - 自动模式：截图后立即使用预设提示词分析
   - 手动模式：截图添加到附件，可编辑后发送

3. **自动提示词**（已预设）：
   ```
   这是一个面试题目的截图。请：
   1. 识别题目类型（算法/系统设计/行为/技术等）
   2. 分析题目要求和考察点
   3. 提供详细的解题思路或参考答案
   4. 如果有代码题，给出清晰的代码实现和复杂度分析
   ```

### 2. 使用截图功能

#### 方式一：点击截图按钮
1. 在面试界面点击工具栏的 📷 图标
2. 如果是选区模式：
   - 屏幕变暗，出现遮罩层
   - 按住鼠标拖动选择题目区域
   - 松开鼠标自动捕获
3. AI 自动分析并给出题解

#### 方式二：使用快捷键
- 默认快捷键可在 **设置 > 快捷键** 中查看/修改

## 故障排查

### 问题 1：截图按钮灰色/无法点击

**原因**：AI Provider 的图片支持检测问题

**解决**：
1. 确保已配置 AI Provider（如 OpenAI、Claude、智谱等）
2. 检查 `localStorage` 中的 `supports_images` 是否为 `true`
3. 如果使用的是国内服务商，本次更新已修复检测逻辑

### 问题 2：点击截图没有反应

**排查步骤**：

1. **检查权限**（macOS）：
   - 前往「系统设置 > 隐私与安全性 > 屏幕与系统音频录制」
   - 确保 Pluely 已添加并启用
   - 重启应用

2. **检查配置**：
   - 设置 > 截图 > 确保已启用
   - 检查是否选择了"选区模式"或"截图模式"

3. **查看日志**：
   - 打开 DevTools（F12 或 Cmd+Option+I）
   - 查看 Console 是否有错误信息

### 问题 3：选区后没有 AI 回复

**可能原因**：
1. AI Provider 未正确配置
2. 自动提示词为空
3. 网络问题

**解决**：
1. 检查设置中的 AI Provider 配置
2. 确保"自动提示词"已填写
3. 切换到手动模式测试：截图后查看附件中是否有图片

### 问题 4：遮罩层无法关闭

**解决**：
- 按 `ESC` 键取消
- 或点击"取消"按钮
- 如果卡死，使用快捷键 `Cmd+Q` / `Alt+F4` 关闭应用

## 最佳实践

### 面试场景推荐配置

```json
{
  "enabled": false,        // 选区模式
  "mode": "auto",          // 自动分析
  "autoPrompt": "这是一个面试题目的截图..."
}
```

### 多题目连续截图

使用**手动模式**：
1. 截图后图片添加到附件
2. 可以继续截取多张
3. 一次性发送给 AI 分析

### 自定义提示词模板

根据不同的面试类型，可以切换提示词：

**算法题**：
```
这是一道算法面试题。请：
1. 分析题目要求和约束条件
2. 提供解题思路（暴力解→优化解）
3. 给出最优解的代码实现
4. 分析时间和空间复杂度
```

**系统设计题**：
```
这是一道系统设计面试题。请：
1. 分析系统需求和核心功能
2. 设计整体架构
3. 说明关键组件和交互
4. 讨论扩展性和容错方案
```

**前端/React 题**：
```
这是一道前端/React 面试题。请：
1. 分析题目考察的知识点
2. 提供实现方案
3. 说明最佳实践和注意事项
4. 如果有代码，指出潜在问题和优化点
```

## 技术细节

### 截图流程

```
用户点击截图按钮
    ↓
检查权限（macOS）
    ↓
根据配置选择模式：
  - 截图模式：直接全屏截图
  - 选区模式：打开遮罩层
    ↓
选区模式：用户拖动选择区域
    ↓
后端裁剪并编码为 base64
    ↓
发送 "captured-selection" 事件
    ↓
前端接收图片，根据模式处理：
  - 自动模式：直接调用 AI
  - 手动模式：添加到附件
```
