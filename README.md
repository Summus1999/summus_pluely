# 功能介绍

## 隐形模式

Pluely 在敏感场景下完全隐形运行。应用采用半透明悬浮窗设计，始终置顶于所有窗口之上，在视频会议、屏幕共享和录屏中完全不可见。窗口经过特殊设计，在 Zoom、Google Meet、Microsoft Teams、钉钉、腾讯会议等主流会议平台中都能完美隐藏，不会被截图捕捉。

当你共享屏幕或录制时，Pluely 对观众完全隐形，同时为你提供实时 AI 辅助。

## 系统音频捕获

实时捕获并转录电脑输出的音频。系统音频功能可直接录制电脑播放的声音，适用于会议、演讲或任何播放中的音频。捕获的音频会发送到你选择的语音转文字服务商处理，可自动发送给 AI 分析或仅做转录。

**快捷键：** `Cmd+Shift+M` (macOS) / `Ctrl+Shift+M` (Windows/Linux)

系统音频捕获包含语音活动检测、实时音频可视化和自动处理状态指示。在音频设置页面配置你偏好的音频输入设备。

## 语音输入

录制语音并使用先进的语音识别技术转为文字。语音输入功能支持多种 STT 服务商，包括 OpenAI Whisper、ElevenLabs、Groq Whisper 及自定义服务商。语音活动检测自动识别说话时机并相应处理音频。

**快捷键：** `Cmd+Shift+A` (macOS) / `Ctrl+Shift+A` (Windows/Linux)

语音输入可在主悬浮窗或聊天对话中使用，实现与 AI 的无声交互。

## 截图捕获

捕获屏幕截图并发送给 AI 进行视觉分析。Pluely 提供两种截图模式：

**全屏截图：** 一键捕获整个屏幕，即刻完成。

**区域截图：** 点击并拖动选择屏幕特定区域进行捕获，精准控制分析范围。

**快捷键：** `Cmd+Shift+S` (macOS) / `Ctrl+Shift+S` (Windows/Linux)

**处理模式：**

**手动模式：** 截图自动添加到附件列表，可捕获多张后统一提交。完全控制截图发送时机。

**自动模式：** 截图自动发送给 AI，使用你预设的提示词。每次自动分析一张截图，即时获取 AI 分析结果。

在截图设置页面配置你喜欢的截图模式和处理方式。

## 文件附件

为 AI 对话附加文件进行分析、审阅或提供上下文。Pluely 支持同时附加多个文件，可发送文档、图片、代码文件或任何文本内容给 AI。文件以卡片形式展示，附带文件类型标识，可单独移除或一键清空。

直接将文件拖入输入区域，或使用附件按钮从系统选择文件。附件会与文字提示一起发送，为 AI 回复提供上下文。

# Pluely 控制台

Pluely 控制台通过直观的侧边栏导航提供对所有功能和设置的全面访问。使用快捷键 `Cmd+Shift+D` (macOS) / `Ctrl+Shift+D` (Windows/Linux) 打开控制台。

## 控制台主页

主控制台显示 Pluely 许可状态和用量统计。激活许可密钥可解锁更快的 AI 响应、高级功能和优先支持。查看 Token 使用情况和 API 活动趋势，通过详细图表了解每日消耗模式。刷新活动数据查看实时的 API 使用情况。

## 对话历史

查看和管理所有对话记录。对话按日期组织，支持按标题搜索和查看消息数量。每个对话显示消息数和最后更新时间。搜索对话快速找到特定主题或讨论。

**查看对话：** 点击任意对话打开详细视图，你可以：

- 阅读完整的消息历史和时间戳
- 发送新消息继续对话
- 附加文件、截图和语音记录扩展对话
- 将整个对话导出为 Markdown 文件
- 在悬浮窗中打开对话供快速参考
- 删除不再需要的对话

视图页面支持连续对话，让你与 AI 保持上下文连贯。所有消息显示用户和 AI 头像、格式化排版和按日期分组的时间戳。

## 系统提示词

创建、管理和组织自定义系统提示词，控制 AI 行为。系统提示词定义 AI 在所有交互中的响应方式和行为特征。页面以网格布局展示所有提示词，支持搜索快速找到特定提示词。

**功能：**

- 创建新的系统提示词，自定义名称和指令
- 编辑现有提示词以优化 AI 行为
- 删除不再需要的提示词
- 选择激活的提示词，用于所有 AI 交互
- 按名称或内容搜索提示词
- AI 辅助生成提示词，帮助创建高效的系统提示词
- 视觉指示器显示当前选中的提示词

每个提示词卡片显示名称、内容预览、创建日期和操作菜单（编辑/删除）。选中的提示词以绿色勾选标记和彩色边框高亮显示。

## 应用设置

配置核心应用行为和外观：

**主题设置：** 在浅色、深色和系统主题间切换。主题全局应用于整个应用，包括悬浮窗和控制台。

**开机自启：** 启用或禁用系统启动时自动运行。启用后，Pluely 在后台静默启动，随时可用。

**应用图标可见性：** 控制 Pluely 图标是否显示在程序坞或任务栏。在敏感场景下隐藏图标以获得最大隐身效果，或保持可见以便快速访问。

**始终置顶模式：** 控制悬浮窗是否始终置顶于其他应用。启用后，无论使用哪个应用，Pluely 都保持可见。禁用则让窗口像普通应用一样行为。

## 回复设置

自定义 AI 生成和显示回复的方式：

**回复长度：** 控制 AI 回复的详细程度。可选：

- 简短：仅包含关键信息的简洁回答
- 适中：平衡的回答，适度详细
- 详细：全面的回答，包含详尽解释
- 自动：让 AI 根据问题复杂度自行决定

**回复语言：** 从 50+ 支持语言中选择 AI 回复语言。语言设置全局应用于所有服务商和对话。语言支持可能因所选 LLM 服务商而异。

**自动滚动控制：** 启用或禁用自动滚动到最新消息。启用时，聊天视图会自动滚动显示新回复。禁用以保持当前滚动位置。

## 截图设置

以精细控制配置截图捕获行为：

**捕获方式：**

- 全屏模式：一键快速捕获整个屏幕
- 区域模式：点击并拖动选择特定区域捕获

**处理模式：**

- 手动模式：截图捕获后添加到附件列表供后续提交。可捕获多张后统一提交。
- 自动模式：截图自动发送给 AI，使用你预设的提示词。配置用于即时分析的自动提示词。

**自动提示词配置：** 选择自动模式时，指定用于自动截图分析的默认提示词。此提示词会随截图一起发送给 AI，无需手动输入。

## 音频设置

配置语音交互和系统音频捕获的输入输出设备。选择偏好的麦克风用于语音录制，配置系统音频捕获设备。页面显示系统检测到的所有可用音频设备及实时设备状态。

## 光标与快捷键

管理光标可见性和自定义键盘快捷键：

**光标设置：**

- 隐形：鼠标悬停 Pluely 时完全隐藏光标（Linux 不支持）
- 默认：标准光标外观
- 自动：光标根据上下文自动变化（指针、文本光标或默认）

**键盘快捷键：** 自定义所有全局快捷键：

- **切换控制台：** 打开或关闭控制台窗口（默认：`Cmd+Shift+D` / `Ctrl+Shift+D`）
- **切换窗口：** 显示或隐藏主悬浮窗（默认：`Cmd+\` / `Ctrl+\`）
- **聚焦输入框：** 将 Pluely 置前并聚焦输入区域（默认：`Cmd+Shift+I` / `Ctrl+Shift+I`）
- **移动窗口：** 按住修饰键并使用方向键连续移动悬浮窗（默认：`Cmd` / `Ctrl`）
- **系统音频：** 开启或关闭系统音频捕获（默认：`Cmd+Shift+M` / `Ctrl+Shift+M`）
- **语音输入：** 开始语音录制进行语音转文字（默认：`Cmd+Shift+A` / `Ctrl+Shift+A`）
- **截图：** 使用配置的模式捕获截图（默认：`Cmd+Shift+S` / `Ctrl+Shift+S`）

所有快捷键可自定义为你偏好的组合。快捷键管理器显示当前绑定，并允许通过按下所需键组合录制新快捷键。

## 开发者空间

开发者空间为开发者和高级用户提供高级配置，集成自定义 AI 服务商和语音转文字服务。

**AI 服务商：**

从全面的预配置服务商列表配置 AI 语言模型服务商，或创建自定义服务商。支持的服务商包括 OpenAI、Anthropic Claude、Google Gemini、xAI Grok、Mistral AI、Cohere、Perplexity、Groq 和 Ollama。

**自定义 AI 服务商设置：**

使用 curl 命令添加任意 LLM 服务商。Pluely 支持完整的流式和非流式功能：

1. 输入包含 API 端点和请求结构的 curl 命令
2. 使用 Bearer Token、API 密钥或自定义头部配置认证
3. 指定响应路径以从 API 响应中提取内容（如 `choices[0].message.content`）
4. 开启流式支持以实现实时响应生成
5. 使用 `{{VARIABLE_NAME}}` 格式在 curl 命令中创建自定义变量

**动态变量：** Pluely 自动替换请求中的这些变量：

- `{{TEXT}}` - 用户文字输入
- `{{IMAGE}}` - Base64 编码的图片数据
- `{{SYSTEM_PROMPT}}` - 系统指令
- `{{MODEL}}` - AI 模型名称
- `{{API_KEY}}` - API 认证密钥

**自定义服务商示例：**

```bash
curl -X POST https://api.example.com/v1/chat/completions \
  -H "Authorization: Bearer {{API_KEY}}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "{{MODEL}}",
    "messages": [
      {"role": "system", "content": "{{SYSTEM_PROMPT}}"},
      {"role": "user", "content": "{{TEXT}}"}
    ]
  }'
```

**STT 服务商：**

从预配置服务商配置语音识别服务商，包括 OpenAI Whisper、ElevenLabs STT、Groq Whisper、Google Speech-to-Text、Deepgram、Azure Speech-to-Text、Speechmatics、Rev.ai 和 IBM Watson STT。

**自定义 STT 服务商设置：**

使用 curl 命令添加任意语音识别服务商：

1. 输入包含 API 端点和音频数据结构的 curl 命令
2. 配置认证和头部
3. 指定响应路径以提取转录文字（如 `text` 或 `results[0].transcript`）
4. 支持各种音频格式和采样率
5. 创建自定义变量实现灵活配置

**STT 动态变量：**

- `{{AUDIO}}` - 音频文件或二进制数据
- `{{API_KEY}}` - API 认证密钥
- `{{LANGUAGE}}` - 目标转录语言
- 你在 curl 命令中定义的自定义变量

**自定义 STT 服务商示例：**

```bash
curl -X POST https://api.example.com/v1/audio/transcriptions \
  -H "Authorization: Bearer {{API_KEY}}" \
  -H "Content-Type: multipart/form-data" \
  -F "file={{AUDIO}}" \
  -F "model=whisper-1"
```

开发者空间提供完全灵活性，可集成任何暴露 REST API 的 AI 或 STT 服务，超越预配置服务商的限制。

## 为何选择 Pluely？

Pluely 重新定义了 AI 助手的可能性。从底层构建就以隐私、性能和隐蔽性为核心，提供企业级能力而不损害数据安全或系统资源。

## 完全隐形

Pluely 以绝对隐秘方式运行。应用采用半透明悬浮窗设计，无缝置顶于所有其他应用之上，除你之外无人可见。

**无处不在的隐形：**

悬浮窗经过特殊设计，在视频会议、屏幕共享、录制和截图中完全不可见。当你在 Zoom、Google Meet、Microsoft Teams 或任何其他会议平台共享屏幕时，Pluely 对观众完全隐藏。半透明设计使其极难被截图捕捉，也不会出现在投影仪或共享屏幕上。

**完美隐身设计：**

窗口透明度可按你的偏好调整，让你既能透过它看到后面，又保持功能完整。可使用快捷键即时显示或隐藏窗口，使用方向键将窗口移动到屏幕任意位置。始终置顶模式确保你需要时它就在那里，而隐藏功能让它在你不需要时完全消失。

**关键使用场景：**

在求职面试中自信使用 Pluely，实时获取信息而不被发现。在销售电话中即时访问产品详情，同时保持专业形象。在技术会议中引用文档而不打断思路。在学习演示期间获取完全对观众隐形的学习辅助。在设计评审期间分析截图并获取建议，无人知晓。在实时代码会话中调试代码并获取语法帮助，完全隐秘。

## 隐私优先架构

你的数据只属于你。Pluely 将隐私作为基础而非事后考虑进行工程设计。

**本地存储：**

所有对话都存储在你设备的 SQLite 数据库中。聊天记录、消息和附件永远不会离开你的电脑。数据库存储在应用数据目录中，具有完整的事务安全和数据完整性检查。你的对话可随时导出为 Markdown 文件，也可随时删除。

**设置和配置：**

所有应用设置、AI 服务商配置、自定义系统提示词、键盘快捷键和偏好设置都存储在浏览器的 localStorage 中。这些数据保留在你的设备上，不会传输到任何地方。你对自己的配置数据拥有完全控制权。

**安全凭证：**

许可密钥和敏感凭证以加密形式存储在应用数据目录的安全存储中，与其他应用数据分离。AI 服务商的 API 密钥存储在 localStorage 中，除直接发送给你选择的 AI 服务商外，不会发送到任何服务器。

**零服务器依赖：**

Pluely 直接从你的设备向你选择的 AI 服务商发起 API 调用。没有代理服务器，没有中间件，没有数据收集。你的对话直接从你的设备流向 OpenAI、Anthropic、Google 或你选择的服务商。你可在浏览器的开发者工具中检查每个网络请求以验证此声明。

**无遥测：**

Pluely 没有分析、没有使用追踪、没有数据收集、没有任何形式的遥测。你的使用模式、对话和行为完全私密。应用不会回传数据、不会报告统计信息、不会收集任何关于你如何使用它的信息。

**离线能力：**

应用对所有本地功能支持离线工作。仅在向 AI 服务商发起 API 调用获取回复时需要互联网。其他所有功能，包括界面、设置、对话历史和系统提示词，完全离线工作。

## 极速性能

基于 Tauri 和 Rust 构建，Pluely 提供原生桌面性能，让基于网页的替代品相形见绌。

**轻量二进制：**

整个应用仅约 10MB，比原版 Cluely 小 27 倍，比基于 Electron 的替代品小得多。尽管体积小巧，它包含完整的 React 前端、Rust 后端、SQLite 数据库和所有功能。

**瞬间启动：**

Pluely 在 100 毫秒内启动。没有启动画面，没有加载转圈，无需等待。点击图标即刻就绪。这种即时启动使其非常适合快速查询和即兴辅助。

**原生性能：**

基于 Tauri，Pluely 作为原生应用运行，使用你系统的 WebView。没有嵌入式 Chromium，没有浏览器开销，没有不必要的资源消耗。比 Electron 应用节省 50% 内存，即使在活跃使用时也几乎没有 CPU 占用。

**高效资源使用：**

应用在正常运行时通常使用不到 50MB 内存。系统音频捕获、语音录制和截图处理都经过性能优化。多个对话、附加文件和聊天记录不会拖慢应用。

**跨平台：**

Pluely 在 macOS、Windows 和 Linux 上原生运行，具有平台特定优化。同一代码库在三个平台上都提供最佳性能，使用原生系统 API 并遵循平台惯例。

## 完全掌控

掌控你的 AI 体验。Pluely 让你对应用的各个方面拥有前所未有的控制。

**任意 AI 服务商：**

使用简单的 curl 命令连接到任意 LLM 服务商。OpenAI、Anthropic、Google、xAI、Mistral、Cohere、Perplexity、Groq、Ollama，或你自己的自定义端点。切换服务商而不会丢失聊天记录或配置。针对不同使用场景使用多个服务商。

**任意 STT 服务商：**

使用 curl 命令集成任意语音识别服务。OpenAI Whisper、ElevenLabs、Groq、Deepgram、Azure、Google，或自定义服务商。完全控制音频格式、采样率和处理参数。即时测试服务商，找到最适合你的声音和语言的最佳识别准确度。

**自定义系统提示词：**

创建无限的系统提示词来控制 AI 行为。定义角色、写作风格、回复格式和专业领域。即时切换提示词，让 AI 适应不同场景。使用 AI 辅助生成功能自动创建高效的提示词。

**灵活配置：**

自定义所有操作的键盘快捷键。调整窗口透明度和始终置顶行为。配置截图捕获模式和处理选项。设置回复长度、语言和自动滚动偏好。选择音频输入设备和捕获设置。一切皆可配置，以匹配你的工作流。

**开源：**

整个代码库以 GPL v3 开源。你可以检查每一行代码、验证隐私声明、审计安全措施，并修改应用以满足你的需求。自行构建、贡献改进，或为特定需求进行分叉。

## 时刻就绪

Pluely 静静地待在桌面，消耗极少资源，同时保持即时可访问。

**一键访问：**

使用键盘快捷键即时显示或隐藏窗口、打开控制台、开始语音录制、捕获截图或切换系统音频。悬浮窗在你需要时随时可用，不需要时隐于无形。

**持久历史：**

所有对话都以完整上下文保存在 SQLite 中。随时返回之前的对话，从断点继续，搜索你的历史。将对话导出为 Markdown 用于文档或参考。

**后台运行：**

Pluely 可在后台静默运行，悬浮窗隐藏。启用开机自启，让其在系统启动时自动运行。隐藏程序坞图标以获得最大隐身效果，同时保持应用运行并通过键盘快捷键访问。

**零维护：**

无需管理订阅，无需维护账户，除了你的 AI 服务商外无需配置任何服务。一旦设置完成，Pluely 就能正常工作。更新在可用时自动推送，你控制何时安装。

## 安装与设置

### 前置条件

- **Node.js** (v18 或更高版本)
- **Rust** (最新稳定版)
- **npm** 或 **yarn**

### 快速开始

```bash
# 克隆仓库
git clone https://github.com/iamsrikanthnani/pluely.git
cd pluely

# 安装依赖
npm install

# 启动开发服务器
npm run tauri dev
```

### 生产构建

```bash
# 构建应用
npm run tauri build
```

这会在 `src-tauri/target/release/bundle/` 中创建平台特定的安装包：

- **macOS**: `.dmg`
- **Windows**: `.msi`, `.exe`
- **Linux**: `.deb`, `.rpm`, `.AppImage`
